## 검증 체크리스트

- [ ] 키보드만으로 첫 스테이지를 끝까지 플레이할 수 있다.
- [ ] 각 직업은 전투 또는 탐험에서 최소 1번 필수적인 순간이 있다.
- [ ] 캐릭터 교체 후 조작권이 0.2초 안에 넘어온다.
- [ ] 피격 원인과 사망 원인을 화면에서 이해할 수 있다.
- [ ] 스테이지 실패 후 10초 안에 재도전할 수 있다.
- [ ] 보스전은 한 직업만 반복해도 쉽게 이기지 못한다.

## 다음 기획 결정

엔진 선택이 가장 먼저 필요합니다.

권장안: **Godot 4**

이유:

- 2D 횡스크롤 액션 제작 속도가 빠릅니다.
- 씬 단위로 캐릭터, 적, 스테이지를 분리하기 쉽습니다.
- 픽셀 아트 카메라와 타일맵 워크플로가 가볍습니다.
- 무료이고 배포 제약이 적습니다.

대안:

- Unity 2D: 에셋 생태계는 좋지만 프로젝트 무게가 커집니다.
- 웹 캔버스: 배포는 쉽지만 액션 RPG 툴링을 직접 만들어야 합니다.
